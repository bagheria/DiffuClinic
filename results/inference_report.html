3. Acceleration Methods

LLaDA: dLLM-Cache

What it does: Caches transformer block activations across adjacent diffusion steps. Only re-computes blocks where tokens changed (transfer_ratio=0.25).

~5× speedup ~15s → ~3s per iteration

Why necessary: Diffusion models run full forward passes for each denoising step (512 per sample). Without caching this is prohibitively slow.

LLaMA: Flash Attention 2

What it does: Fuses attention operations in GPU SRAM, avoiding materializing the full N×N attention matrix in HBM.

<1% speedup here

Why minimal: At ~2,000 token context, the attention matrix is only ~5M elements — fits comfortably in GPU cache. Flash Attention shows its real benefit at 8K–32K+ sequence lengths.
