Safety — Factuality vs Source Text

核心问题：总结有没有编造原文没说的东西？
QAFactEval
QA-based (Question Answering)
从总结中抽取关键实体 → 生成问题 → 回原文找答案 → 比对一致性
擅长抓：实体替换、数值错误、药物剂量偏差
SummaC
NLI-based (Natural Language Inference)
将总结逐句切分 → 用 RoBERTa-MNLI 判断每句是否被原文蕴含
擅长抓：无依据断言、逻辑矛盾、与原文不符的推理
MedNER F1
NER-based (Named Entity Recognition)
分别从原文和总结中提取医疗实体 → 计算 Precision / Recall / F1
擅长抓：药名遗漏/篡改、剂量/检查项/诊断实体丢失
互补机制：QA 检测实体级替换，NLI 检测句子级矛盾，NER 检测领域实体保留。 QAFactEval 原论文证明 QA+NLI 结合优于任一单独（+14% on SummaC benchmark）。 WeCheck (ACL 2023) 验证联合弱标注 > 单一信号。
2

Quality — Relevance vs Reference Summary

核心问题：生成的总结和参考总结有多像？
ROUGE-1/2/L
N-gram Overlap (Lexical)
unigram, bigram, and longest common subsequence F1 against reference
BLEU-1 .. BLEU-4
Precision-based N-gram Match
1-gram through 4-gram precision with brevity penalty and smoothing
METEOR
Word Alignment (Lexical + Stemming)
Unigram precision + recall with stemming and synonym matching
BERTScore-F1
Embedding Similarity (Semantic)
Token-level cosine similarity using roberta-large, rescaled with baseline
互补机制：从纯词汇重叠 (ROUGE/BLEU) 到词对齐 (METEOR) 到语义相似 (BERTScore)， 避免单一粒度对语义等价表达的不公平惩罚。

决策逻辑 (Decision Rule)

当模型在各维度 trade-off 时，按优先级决策： 

Model A: Safety ✓✓✓  Quality ✓✓  Clinical ✓  Efficiency ✓
Model B: Safety ✓    Quality ✓✓✓  Clinical ✓✓✓  Efficiency ✓✓✓

→ 选 Model A。Safety > Everything.
